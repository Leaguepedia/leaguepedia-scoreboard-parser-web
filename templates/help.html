Parser usage help

This website takes Riot game IDs or QQ match IDs (used in the Chinese server) and generates scoreboards for them. The output scoreboard text is intended to be pasted on Leaguepedia.

Important information:
  • Remember that the IDs must be entered in Data before parsing scoreboards for them, otherwise the parser will throw an error.
  • Do NOT change anything (player names, items, tournament, etc.) in the output text. If one of the players has a red link you need to create a redirect.
  • If the parser returns warnings below the output, check if all the players are already in the participants section of the tournament, and click Clear Cache. If it still fails you can ask for help on Discord, but don't copy scoreboards in the wiki if there are warnings you don't understand.
  • If you get warnings that you already fixed, click Clear Cache.
  • If you are parsing QQ IDs, keep in mind that the patch is taken from Data, so make sure the patch is correctly entered there.
  • Ask on Discord if you are unsure about something.

IDs:

You should fill the game/series IDs here. If you are using Riot game IDs, you need to enter one ID per line, you can also separate multiple series with blank lines. If you are using QQ series IDs, you have to leave one blank line between each other.

Usage examples:




Game Source:

If you are parsing QQ IDs, select QQ. Otherwise, select Riot if the games were played on the tournament realm (in this case the ids start with ESPORTSTMNT or LOLTMNT), or Riot Live if they were played on a live server. Most times Riot is the correct option. Your selection will be remembered the next time you open the page.

Include header:

Whether to include the series header in each parsed series. Only check this option if you are parsing a Best of 1 or the first game of series.
